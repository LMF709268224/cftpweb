# 2026-05-30 微服务接口 Web 接入与联调总结

## 本次接入范围

本次主要跟进 GLMS / GCC / gcreds / gmsg 新版接口变化，补齐 adminserver 与 candidateserver 的 BFF 和 Web 端可测试入口。

## adminserver - 管理端课程/管线配置

1. GLMS 课程管理页 `/lms`
   - 课程列表继续使用新版分页接口：`GET /api/lms/courses?page_size&page_token&category_tips&published_only`。
   - 课程基础信息继续支持创建、保存、发布、删除草稿版本。
   - 课程封面上传继续使用新版上传协议：前端计算 SHA-256，调用 `/api/lms/upload-url` 传 `file_hash`，上传 S3 时只使用后端返回的 `signed_headers`。
   - 导入 JSON 保留课程导入和课检导入两个入口：
     - 课程导入：`POST /api/lms/import`，`scope=course`
     - 课检导入：`POST /api/lms/import`，`scope=quiz`
   - 新增“加载课程统计”按钮，接入 `GET /api/lms/courses/{course_id}/detail`，用于验证新版 `GetCourseDetail`。
   - “完整课程预览”继续接入 `GET /api/lms/courses/{course_id}/complete`。

2. GLMS 报名和学习进度联调
   - 报名列表接入：`GET /api/lms/courses/{course_id}/enrollments`。
   - 每条报名新增多个测试操作：
     - “详情”：`GET /api/lms/enrollments/{enrollment_id}`
     - “进度”：`GET /api/lms/courses/{course_id}/candidates/{candidate_id}/progress`
     - “课时进度”：`GET /api/lms/lesson-progress?candidate_id=...`
     - 课时进度里的“详情”：`GET /api/lms/lessons/{lesson_id}/progress?candidate_id=...`
     - “章节进度”：`GET /api/lms/chapters/{chapter_id}/progress?candidate_id=...`
     - “课检尝试”：`GET /api/lms/quizzes/{quiz_id}/attempts?candidate_id=...`
     - 课检尝试里的“详情”：`GET /api/lms/quiz-attempts/{attempt_id}`
   - 注意：章节进度需要先在课程内容里选中一个章节；课检尝试需要先选中一个课检。

3. GLMS 课程资源联调
   - 原“异常资源”区域改为“课程资源”，接入新版 `GET /api/lms/assets`。
   - 支持资源状态筛选：全部 / 正常 / 异常 / 缺失。
   - 支持资源类型筛选：全部 / 封面 / 资料 / 课时。
   - 每条资源可点“详情”，接入 `GET /api/lms/assets/detail?object_key=...&associated_id=...`。

4. GCC 管线配置页 `/pipelines`
   - 已接入 GCC 管线列表、创建、结构配置、发布、删除和 GLMS 课程单元关联。
   - GLMS 课程下拉使用已发布课程列表，方便把一个管线关联到多个课程。

## candidateserver - 考生端课程/管线展示

1. 课程中心 `/courses`
   - “全部管线”接入 `GET /api/mall/pipelines`。
   - 展示真实管线阶段数、课程单元数、结业资格数、支付配置状态和分类提示。
   - “我的管线”继续使用 `GET /api/pipeline`。
   - “学习资料”继续使用 `GET /api/pipeline/materials`。

2. 管线详情 `/courses/detail?id=...`
   - 接入 `GET /api/mall/pipelines/{pipeline_id}` 展示管线详情。
   - 新增考生侧只读课程摘要接口：`GET /api/mall/courses/{course_id}`。
   - 管线单元不再只显示 GLMS course id，会额外显示课程标题、分类和时长，便于验证 admin 端配置的课程是否正确出现在 candidate 端。

3. 购买入口
   - 仍使用 `POST /api/mall/pipelines/{pipeline_id}/purchase`。
   - 当前显示的是 Stripe Product ID / Price ID 是否已配置，不直接展示金额。

## 仍需关注或等待微服务的点

1. GCC catalog/category 管理接口目前 proto 里仍缺失，所以分类管理相关 TODO 保留。
2. candidate 端真正“进入 GLMS 学习、完成课时、参加课检、提交课检”的页面还未完整产品化；本次先把管线和课程可视化联调接上。
3. gcreds 新版 summary 不再返回证书文件列表，candidate 证书页文件区域暂不展示文件详情。
4. gmsg 新版消息列表不再携带 message payload，candidate 消息页暂按模板内容兜底展示。

## 联调建议

1. 管理端 `/lms`
   - 创建或导入一门课程。
   - 上传封面，确认 Network 中 `/api/lms/upload-url` 请求包含 `file_hash`，S3 PUT 使用返回的 `signed_headers`。
   - 创建章节、文本课时、课检、题目和选项。
   - 发布课程。
   - 点击“加载课程统计”和“加载完整结构”确认接口返回。

2. 管理端 `/pipelines`
   - 新建管线。
   - 在课程单元下拉中选择已发布 GLMS 课程。
   - 配置 Stripe Product ID / Price ID。
   - 保存结构并发布。

3. 考生端 `/courses`
   - 查看全部管线是否出现刚发布的管线。
   - 进入详情页，确认管线单元显示课程标题，而不是只显示 course id。
   - 点击购买入口，检查 payment url 或订单接口响应。
