# CFtP 系统微服务总览

本项目采用了微服务架构，后端各个功能模块被拆分为了多个独立的 Go 微服务。微服务之间主要通过 **gRPC** 进行内部通信，部分微服务通过订阅 **NATS** 消息中间件处理异步事件。对外则通过 BFF (Backend For Frontend) 层如 `candidateserver` 将聚合后的 RESTful API 提供给前端。

以下是项目中所有主要微服务的职责定位以及对外提供的接口总览。

---

## 1. 基础架构与公共服务

### **cfgserver (配置中心服务)**
- **作用**：负责集中管理和分发所有微服务的配置。通过云厂商 KMS (如阿里 KMS) 对系统配置进行加密(TOML 格式)，并在微服务启动时通过 gRPC 将解密后的配置下发给各服务。
- **提供的 gRPC 接口 (`ConfigService`)**：
  - `GetSystemConfig`: 提供给各微服务拉取自己的运行配置。

### **gmid (身份映射解析服务)**
- **作用**：用于对接 IAM (Casdoor)。负责将外部身份系统生成的 UUID 与内部业务系统使用的 ULID 进行互相转换与映射。
- **提供的 gRPC 接口 (`MidService`)**：
  - `GetUlidByUUID`: 根据 Casdoor 的 UUID 获取业务 ULID。
  - `GetUUIDByUlid`: 根据业务 ULID 获取对应的 UUID。

### **gmsg (用户消息/站内信服务)**
- **作用**：负责管理发给考生的站内信消息。采用“数据与表现分离”的设计思想，数据库仅记录发送关系与核心 JSON 参数，前端通过模板 ID 动态渲染内容。
- **提供的 gRPC 接口 (`MessageService`)**：
  - `ListMessages`, `MarkAsRead`, `DeleteMessages`, `ClearAllMessages`: 供考生查询和管理消息。
  - `SendMessage`, `RevokeMessage`: 供其他服务内部调用，发送或撤回消息。
  - `CreateTemplate`, `GetTemplate`, `UpdateTemplate`, `ListTemplates`: 消息模板的管理。

### **gmail (邮件发送服务)**
- **作用**：基于 go-mail 和企业邮箱/SMTP（如 Mailgun），响应其他业务系统的邮件发送请求。支持使用 Go `text/template` 或 `html/template` 进行模板渲染。
- **提供的 gRPC 接口 (`MailService`)**：
  - `CreateMail`, `GetMailStatus`, `CancelMail`: 创建、查询、取消邮件发送任务。
  - `CreateTemplate`, `UpdateTemplate`, `GetTemplateList`, `GetTemplate`: 管理邮件模板。

### **gpdf (PDF 生成服务)**
- **作用**：通用的 PDF 生成引擎。主要用于在考生通过认证后生成电子证书。
- **提供的 gRPC 接口 (`PdfGenService`)**：
  - `GenPdf`: 接收渲染参数并生成 PDF。

## 2. 核心业务服务 (商城、课程与认证)

### **gcc (Global Course Catalog - 课程与认证配置服务)**
- **作用**：静态课程、考试与认证管线 (Pipeline) 的中央配置库。存储所有版本化的管线阶段 (Stage) 和原子单元 (Unit) 信息。
- **提供的 gRPC 接口 (`CCService`)**：
  - `CreatePipelineDraft`, `UpdatePipelineStructure`, `PublishPipeline`: 管线版本的编辑与发布。
  - `GetPipeline`, `ListPipelines`: 查询当前生效的管线与课程大纲。
  - `CreateCatalog`, `ListCatalogs`: 课程/认证分类目录的管理。
  - `GetPipelineFinalEligibility`: 获取结业资格等。

### **glms (Learning Management System - 学习资料管理服务)**
- **作用**：管理具体的课程学习资料（视频、PDF、试卷等）。配合 Casdoor 鉴权体系，在验证用户具备学习权限后，下发实际的资源访问链接。
- **提供的 gRPC 接口 (`LmsService`)**：
  - `CreateCourse`, `UpdateCourse`, `PublishCourse`: 课程与资料的整体管理。
  - `CreateChapter`, `CreateLesson`, `CreateCourseMaterial`: 章节与具体课件资源的管理。
  - `CreateQuiz`, `CreateQuizQuestion`, `CreateQuizOption`: 随堂测验管理。
  - `CreateUploadURL`, `CreateViewURL`: 颁发对象存储 (S3) 的上传与查看凭证。

### **gprog (Progress - 进度与流程引擎服务)**
- **作用**：管理**每个考生具体购买的课程/认证进度**。它是事件驱动的流程引擎，接收外部考试成绩 (gexam) 和学习进度 (glms) 的回调事件，推动考生的“管线”往前滚动。
- **提供的 gRPC 接口 (`ProgService`)**：
  - `PipelineCreate`: 为考生创建实例（由商城 gmall 在支付成功后调用）。
  - `CandidateSignupExam`, `CandidateApplyRetake`: 考生报名考试或申请重考。
  - `ListCandidatePipelines`: 获取考生当前的认证与学习进度。
  - 还有各种管理员干预接口 (`AdminForceCourseCompleted` 等)。

### **gcreds (Credentials - 资格与证书服务)**
- **作用**：处理考生的“资格审查”和“证书颁发”。包括管理考生的免考申请审核，以及满足条件后为考生生成数字证书。
- **提供的 gRPC 接口 (`CredentialService`)**：
  - `SubmitApplication`, `AuditApplication`: 提交与审核免考/认证资格申请。
  - `GetLatestCredential`, `RevokeCredential`: 获取与管理最终证书。
  - `CheckCandidateQualification`: 检查某位考生是否具备解锁某课程/考试的资格。

## 3. 交易、支付与开票系统

### **gmall (商城与订单服务)**
- **作用**：处理课程、认证商品的展示与交易。向 gcc 拉取商品，向 gcreds 校验购买资格，最终生成订单并通知 gprog 履约。
- **提供的 gRPC 接口 (`MallService`)**：
  - `CreatePipelineOrder`, `CreateStageOrder`, `CreateCourseRetakeOrder`: 创建对应的购买订单。
  - `GetPipelineOrderStatus`, `GetStageOrderStatus`: 查询订单状态。

### **gpay & gstripe (支付网关服务)**
- **作用**：`gpay` 是统一的内部支付网关抽象，`gstripe` 则是专门负责与外部 Stripe 对接的具体实现（包含接收 Stripe Webhook 事件并更新订单状态）。
- **提供的 gRPC 接口 (`PaymentService` - gpay提供)**：
  - `CreatePayment`, `GetPaymentStatus`, `CancelPayment`: 供 gmall 调用以发起和查询支付流。

### **ginvoice & gxero (开票系统服务)**
- **作用**：负责对公/个人的发票生成。`gxero` 专门负责与外部 Xero 财务系统对接。
- **提供的 gRPC 接口 (`InvoiceService`)**：
  - `CreateInvoice`, `QueryInvoice`, `DownloadPdf`: 请求生成并发票下载。

## 4. 考试生态与对接 (GEE 集成)

### **gexam (考试系统对接服务)**
- **作用**：考试核心聚合服务。与外部真实考试平台 (如 GEE ƽ̨) 的 API 进行对接。处理成绩拉取、考试预约排期等。
- **提供的 gRPC 接口 (`GExamService`)**：
  - `CreateExam`, `GetExam`: 考试计划的创建与查询。
  - `GetExamResultDetail`: 拉取考试成绩明细。
  - `GetScheduleURL`: 获取考生预约外部考试的跳转链接。
  - `TermUrlCallback`: 处理外部平台的回调逻辑。

### **geex, geewhook, geemock (考试辅助套件)**
- **geex**: 处理 GEE 相关的事件推送、成绩拉取等离线定时任务。
- **geewhook**: 作为独立的 Webhook 服务接收外部 GEE 平台推送的事件（如交卷事件）。包含 MySQL、Redis 分布式锁来保证并发和审计数据的幂等性。
- **geemock**: 开发测试使用的“虚拟 GEE 平台”，提供一个假的前端页面用来模拟考生交卷与发布成绩，让内部流程不依赖真实 GEE 环境即可跑通。

---

## 5. 接入层 (BFF)

### **candidateserver (考生端 BFF 服务)**
- **作用**：直接向 C 端 Web/App 提供 HTTP RESTful 和 GraphQL 服务。
- **特点**：自身没有重度业务逻辑数据库，主要职责是处理 HTTP 请求鉴权（Casdoor Token）、聚合并编排底层各种 `g-` 开头微服务的 gRPC 请求，最后将组装好的 JSON 数据返回给前端。
