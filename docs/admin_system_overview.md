# CFTP 管理系统 (Admin Web) 功能导览与培训概要

## 1. 编写背景与会议目标
客户初次接触 CFTP 管理后台时，由于功能模块较多（涵盖教务、财务、配置、运维等），容易产生“无从下手”的感觉。
本次会议的核心目标：
1. **消除陌生感**：用通俗易懂的语言，按“业务场景”对系统功能进行分类串讲，帮助客户建立全局观。
2. **解答疑惑**：针对客户在试用过程中遇到的具体流程问题（如：如何上架一门课？如何审批资格？）进行解答。
3. **收集反馈与建议**：听取客户在交互体验（UI/UX）、功能缺失、或者操作繁琐度上的意见，为后续系统的迭代优化提供方向。

---

## 2. 核心业务模块概览（按使用场景划分）

为了让客户更容易理解，我们不按菜单的上下顺序，而是按照**实际业务角色和工作流**将系统拆分为以下几个核心板块：

### 📈 A. 日常运营与财务看板（管理层、财务、客服人员）
这部分是日常查看最多、数据最直观的模块。
* **运营看板 (Dashboard)**：一目了然地查看当前的活跃考生、报名情况、今日流水等核心数据。
* **订单管理 (Orders)**：查看考生购买课程、考试或申请认证的订单明细与支付状态。
* **发票管理 (Invoices)**：处理和查看考生的发票请求与支付凭证。

### 🎓 B. 教务与审批中心（教务老师、审核专员）
这部分负责处理考生的学习进度跟踪和资质审批。
* **认证管理 (Pipeline Management)**：跟踪每个考生在某个认证项目（如 CFtA）中的进度（报名 -> 学习 -> 考试 -> 拿证）。
* **考试管理 (Exams)**：查看考生的考试预约、考试成绩以及状态流转。
* **审核中心 (Review Center)**：处理考生提交的资质认证申请（例如工作经验证明审核、免考资格审批等）。

### ⚙️ C. 核心商品与课程配置（教研团队、产品经理）
**这是系统最核心也是最复杂的部分**，主要用于“造轮子”。建议客户在熟悉日常运营后，再深入学习此模块。
* **课程配置 (Course Config) & 资源包**：配置在线学习的课程内容、章节、课时，以及上传相关的讲义、视频等资源文件。
* **认证配置 (Pipeline Config)**：设计一个认证的通关流程（例如：先学完指定课程 -> 再通过某场考试 -> 最后获得证书）。
* **资格定义 (Credential Definitions)**：定义系统里有哪些证书、免考资格或通用资质。
* **商品配置 (Bundle Config)**：把上面配置好的“课程”或“认证”打包成一个商品，定好价格，上架供考生购买。
* **证书模板配置 (Certificate Templates)**：维护考生最终拿到的 PDF 证书的样式和排版。

### ✉️ D. 消息与通知触达（运营、客服人员）
* **站内信 (Messages)**：查看和管理发给系统内考生的站内通知。
* **邮件中心 (Mails)**：管理邮件模板，查看系统自动发送的各类通知邮件到达情况。

### 🛠 E. 系统与运维支持（IT 管理员、技术支持）
* **考生权限管理 (Permissions)**：特殊情况下，人工介入调整某个考生的访问权限。
* **审计日志 (Audit Logs) & Webhook 审计**：追踪系统里的关键操作记录，以及外部系统（如支付网关）的回调情况，主要用于排查问题。
* **事件与任务 (Events & Tasks) & 证书生成流水**：查看后台异步任务的执行状态，比如某批证书有没有成功生成 PDF。

## 3. 建议的新手探索路径 (Onboarding Path)

为了避免客户一上来就迷失在复杂的“配置”中，建议引导他们按照以下路径体验：
1. **先看结果**：去【运营看板】和【订单管理】看看数据是什么样的。
2. **体验日常处理**：去【审核中心】模拟一下同意/拒绝一个考生的申请。
3. **深入底层配置**：最后，带他们走一遍“如何上架一个新证书”的完整闭环（即从“课程配置” -> “认证配置” -> “商品配置”）。
