# CFtP 考生端流程与能力图谱

## 目标
为 CFtP 和 CFTP 构建完整的考生体验流程：购买管线 -> 查看课程结构 -> 打开课程资料 -> 学习并同步进度 -> 申请考试 -> 查看考试结果 -> 查看证书。

## 目前已实现
- 考生可以在 `/courses` 浏览已购买的管线。
- 考生可以打开管线详情并从已购买的课程跳入学习。
- 考生可以打开某门课程的学习中心，并查看章节和课时。
- 考生可以打开外部课时内容，并将其标记为已完成。
- 考生可以通过 `/api/progress/courses/{courseId}/sync` 同步课程进度。
- 考生可以通过 `/exams` 查看考试列表数据。
- 考生可以通过 `/exams/{examId}` 打开考试结果页面。
- 考生可以在 `/certificates` 查看已下载的证书。

## 仍需补充
- 课时级别的进度自动上报仍然比较粗粒度。
- 我们尚未发送周期性的播放位置、滚动进度或文档阅读百分比。
- 考试页面仍然依赖于后端数据的可用性来展示有意义的当前/免考/记录流程。
- 免考申请流程目前仍然是一个后端 TODO。
- 考试列表/结果流程目前只展示返回的 gexam 数据；尚未拥有专门的、对考生友好的考试工作流 UI。
- 证书申领并非第一级操作；今天它仅作为一个列表/查看页面存在。

## 后端能力说明
- `glms.SyncCourseProgress` 已存在并已接入 candidate BFF。
- `glms.CompleteLessonLearning` 已存在并已接入 candidate BFF。
- `gexam.ListExams` 和 `gexam.GetExamResultDetail` 已存在并已接入 candidate BFF。
- `gprog` 仍缺少考生免考申请流程，因此 `/api/exams/units/{courseUnitUlid}/exemption` 仍被阻塞。

## 后续优先级
1. 在学习页面添加真正的细粒度进度上报。
2. 添加带有真实考生专属 UI 的考试报名 / 免考 / 重考工作流。
3. 添加证书申领/资格仪表盘，以便学习者知道证书何时准备就绪。
4. 清理剩余的本地化（locale）文件，并将新的本地页面文案移入 i18n。
