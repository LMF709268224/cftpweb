# 前端订单中心对接指南

本文档旨在说明前端（例如“我的订单”、“购买记录”等模块）与微服务 `gmall` 订单系统对接的标准规范和状态判定逻辑。

## 1. 核心架构设计：列表与详情分离

系统采用“通用支付订单”与“具体业务订单”分离的设计。在用户刚完成下单时，系统会**同步**在通用表和业务表中创建记录。因此，前端在对接时的核心原则是：**列表查通用表，详情查业务表**。

### 1.1 列表页：统一调用通用订单 API (`ListOrders`)
前端在渲染“我的订单”或“购买记录”等聚合列表时，**只需调用通用的 `ListOrders` 接口**。
- 这个接口会返回用户所有的流水单（包含未支付和已支付）。
- > [!IMPORTANT]
  > **关键数据展示**：通用订单对象中有一个 `Meta` 字段（JSON格式），里面直接存有 **`ProductName` (商品名称)**。前端完全可以直接读取这个字段来展示商品名称和价格等关键信息，**不需要**在列表页去额外调用业务订单接口。

### 1.2 详情页：调用具体业务订单 API
当用户在列表页点击某个订单的“查看详情”时，由于不同商品（套餐、课程重修、资格申请）具有不同的特定数据结构，前端需要：
- 读取通用订单上的 `biz_type`（业务类型，如 `BUNDLE_PURCHASE`）和 `biz_ref_ulid`（业务主键 ID）。
- 根据业务类型，去调用相应的业务详情接口（例如 `GetBundleOrder`、`GetStageOrder` 等），以展示具体的商品构成、所选的豁免项或申请资料等业务级明细。

## 2. 状态判定逻辑与按钮交互

每个通用订单 (`Order`) 都包含两个关键的状态字段（全部为大写英文字符串）：
- **`order_status`**: 订单整体的生命周期状态。
- **`payment_status`**: 底层实际支付扣款的状态。

前端页面（尤其是操作按钮的展示）应主要依据这两个字段来进行判断。

### 2.1 【去支付】与【取消订单】按钮
订单在未完成扣款前，允许用户继续支付或主动取消。
- **判断条件**：
  - `order_status` 等于 `"WAIT_PAYMENT"`
  - 或者 `order_status` 等于 `"PENDING"`
- **前端操作**：
  - 展示 **[去支付 / 继续支付]**：如果用户点击，则继续调起结账页面。
  - 展示 **[取消订单]**：如果用户点击，将 `biz_ref_ulid` 传给后端的取消接口。

### 2.2 【查看发票 / 收据】按钮
订单已经成功扣款并且业务权益已经正常发放完毕。
- **判断条件**：
  - `order_status` 等于 `"COMPLETED"`（此状态下 `payment_status` 通常为 `"PAID"`）
- **前端操作**：
  - 展示 **[查看发票 / 收据]**。
  - 可以展示“前往学习/查看内容”等跳转动作。
  - ❌ **锁定操作**：此状态下绝对不可再显示取消或支付按钮。

### 2.3 异常与终结状态处理
订单因为用户主动取消、超时未支付、或者支付失败等原因彻底作废。
- **状态特征**：
  - `order_status` 等于 `"CANCELLED"`（已取消）或 `"CLOSED"`（已关闭）。
- > [!WARNING]
  > **支付失败处理机制**：如果用户的银行卡被拒或 Stripe 明确返回扣款失败，底层的 `payment_status` 会变为 `"FAILED"`。在后端逻辑中，只要支付状态变为 FAILED，系统的**生命周期状态 `order_status` 会被同步强制更新为 `"CANCELLED"`**。
- **前端操作**：
  - 订单已被视为**废弃的终态**，只能作为历史记录展示。
  - 对于这种状态的订单，如果用户想买，只能引导其重新创建一个新订单。
  - ❌ **锁定操作**：隐藏所有的支付、取消等操作按钮。对于支付失败的，可以给一个友好的 UI 提示（“您的支付失败，请重新购买”）。

---

### 💡 附录：核心字段映射清单

| 字段名 | 类型 | 说明/示例 |
| --- | --- | --- |
| `biz_type` | String | `BUNDLE_PURCHASE` (套餐), `STAGE_PAYMENT` (阶段购买) 等 |
| `biz_ref_ulid` | String | 业务单的 ID，**发起支付或取消时必须传的参数** |
| `order_status` | String | `WAIT_PAYMENT`, `PENDING`, `COMPLETED`, `CANCELLED`, `CLOSED` |
| `payment_status` | String | `UNPAID`, `WAIT_PAY`, `PAID`, `FAILED`, `REFUND_OFFLINE` |
| `Meta.ProductName` | String | 商品的展示名称，供前端直接用于列表渲染 |
